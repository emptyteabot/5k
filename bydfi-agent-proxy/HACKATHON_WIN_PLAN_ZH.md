# BYDFI AgentOS 黑客松赢法

## 一句话

这次别把自己讲成做了一个聊天网页。你要讲的是：我做了一个 **BYDFI 定制版 Hermes Agent 网页终端**，它把复杂能力藏在一个极简入口后面，用最少的学习成本，换来最强的视觉冲击和最顺的现场演示。

## 这次路演到底卖什么

不是卖代码干净。
不是卖架构优雅。
不是卖“我们以后可以做很多事”。

这次卖的是三件事：

1. 评委一眼能看懂
2. 主办方一耳朵就知道和 BYDFI 有关系
3. 现场 60 秒内能跑出一个像样的闭环

## 评委真正会买单的点

### 1. 利益对齐

不要讲通用 AI，不要讲空泛平台。
要直接讲：BYDFI 的产品复杂，用户理解成本高，员工协作成本也高。我的系统把复杂意图压成一个网页入口，让员工和用户都能用中文直接发起任务。

### 2. 视觉溢价

同样一套底层能力，放在命令行里像工具，放在网页里像产品。
所以网页不是附属物，而是定价器。

### 3. 零摩擦演示

评委不会原谅转圈、卡顿、API 报错。
现场最重要的不是“全部真实”，而是“关键路径绝对稳”。
真正该硬的地方要硬起来：固定演示词、固定返回节奏、固定展示卡片。

### 4. 裂变想象力

最后不能只是“回答完毕”。
一定要让评委看到：这个东西还能继续带来传播、增长、复用、标准化。

## 现在最适合讲的产品定义

**BYDFI AgentOS = 一个部署在云上的 BYDFI 定制版 Hermes Agent 网页终端。**

表层是一个极简中文输入框。
底层是 Hermes 的记忆、skills、工具调用、持续执行能力，再叠加你自己已经做出来的内部资料、报告流、审计脚本和网页桥接层。

## 不要再讲的东西

- 不要讲“取代整个公司”
- 不要讲“全知全能”
- 不要讲“只是知识库问答”
- 不要讲“未来可能支持一切平台”
- 不要讲太多后端细节

这些不是加分项，只会把评委带到你最弱的地方。

## 要讲的东西

- 这是 BYDFI 定制版，不是通用壳子
- 这是网页入口，不是冷冰冰的脚本
- 这是执行终端，不是单纯总结器
- 这是可部署在云服务器上的系统，不依赖你电脑开着
- 这是有底座的，不是纯前端演示皮

## 唯一必须打中的 Wow Moment

推荐你把高潮功能定义成：

**Activate BYDFI Sentinel**

演示路径固定成下面这 5 步：

1. 输入一句带业务感的自然语言
2. 界面立刻把意图拆成结构化参数
3. 系统展示风险门槛 / 执行判断 / 下一步动作
4. 页面给出一个明显的“已生成结果”状态卡
5. 最后再给一个可分享的战报或摘要卡片

这 5 步的价值不是技术炫技，而是让评委在 30 到 60 秒内形成一个直觉：

“这个东西不是在陪我聊天，它是在替我干活。”

## 这个 Wow Moment 具体长什么样

推荐用这句演示词：

`帮我执行 6 周年策略检查`

推荐页面表现：

- 左侧：输入和对话
- 中间：主结果卡片
- 右侧：意图拆解 / 风险校验 / 执行日志
- 底部：结果状态条或战报卡

推荐系统输出顺序：

1. 识别意图
2. 拆参数
3. 给风险边界
4. 输出动作结果
5. 生成可传播卡片

## 现在仓库里哪些东西已经能支撑这套讲法

### `bydfi-agent-proxy/`

这是当前网页入口。
已经接了 Hermes 桥接层，能正常中文对话，适合作为黑客松展示壳。

### `hermes-agent/`

这是底层引擎仓库。
你不用在路演里展开讲源码，但它是你“不是套皮玩具”的底气来源。

### `bydfi-audit-bot/`

这里是你过去已经沉淀出来的真东西：
采集、报告、脚本、MCP、交付物、管理层 PDF、调度任务。
这部分决定了你不是空讲 Agent，而是已经有企业材料和工作流基础。

### `references/`

这里放了黑客松原始参赛材料，方便解释项目为什么会长成现在这个样子。

## 真能力和演示包装要分开

要分清两层：

### 真能力

- Hermes 接入已经能用
- 网页聊天入口已经能用
- 中文输出已经正常
- 历史 BYDFI 工作流、报告流、脚本和资料都已经归档进仓库

### 演示包装

- 视觉高潮路径要固定
- 部分展示卡片可以为路演专门设计
- 部分状态流可以做成稳态 demo

这不叫造假，这叫把不可控变量隔离出主舞台。

## 最后一句该怎么说

不要说“我们做了一个聊天机器人”。

要说：

**我做的是一个部署在云上的 BYDFI 定制版 Hermes Agent。表面是极简网页，底层是可持续调用资料、记忆和执行能力的企业级 AI 终端。**
